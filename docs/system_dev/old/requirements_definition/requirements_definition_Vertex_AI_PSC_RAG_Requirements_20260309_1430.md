# 要件定義書：Vertex AI PSC RAG システム

## 1. 目的
本システムは、Vertex AI Search を PSC 経由で利用し、社内ドキュメントに対するセキュアな検索・回答機能を提供することを目的とする。

## 2. 機能要件
- **閉域RAG機能**: インターネットを経由せず、PSC 経由でドキュメント検索および LLM 回答を取得する。
- **部署間認可**: ユーザーが所属する `department_id` に基づき、検索対象ドキュメントを厳格に分離する。
- **データ管理**: 管理者は管理画面よりドキュメントをアップロードし、メタデータとして所属部署を付与する。

## 3. 非機能要件（テスト可能な数値目標）
- **性能（応答速度）**:
    - 単純検索レスポンス: 95パーセンタイルで 1.5秒以内。
    - 生成AI回答レスポンス（Streamingなし）: 95パーセンタイルで 5.0秒以内。
    - PSC経由によるオーバーヘッド: 1リクエストあたり 50ms 以下。
- **可用性**:
    - サービス稼働率: 99.9% 以上（月間）。
- **同時接続数**:
    - 最大同時アクティブユーザー数: 100名。

## 4. セキュリティ・制約事項
- **ネットワーク**: インターネットゲートウェイの利用禁止。全ての Google API へのアクセスは PSC エンドポイントを経由すること。
- **暗号化**: データ保存領域（GCS, Vertex AI Search）は、顧客管理暗号鍵（CMEK）を使用すること。
- **API制約（遵守規則）**:
    - `data_store_id` および `engine_id`: `^[a-z0-9-_]{1,63}$` (先頭/末尾は英数字のみ)
    - ドキュメントサイズ: 1ファイルあたり最大 100MB。
    - メタデータキー名: `^[a-zA-Z][a-zA-Z0-9_]{0,63}$`
