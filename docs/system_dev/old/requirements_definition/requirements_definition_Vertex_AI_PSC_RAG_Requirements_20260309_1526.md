# 要件定義書：Vertex AI PSC RAG システム

## 用語定義
- **PSC**: Private Service Connect（Google Cloud 内でマネージドサービスへ閉域接続する仕組み）

## 1. 目的
本システムは、Vertex AI Search を PSC 経由で利用し、社内ドキュメントに対するセキュアな検索・回答機能を提供することを目的とする。

## 2. 機能要件
- **閉域RAG機能**: インターネットを経由せず、PSC 経由でドキュメント検索および LLM 回答を取得する。
- **部署間認可**: ユーザーが所属する `department_id` に基づき、検索対象ドキュメントを厳格に分離する。
- **データ管理**: 管理者は管理画面よりドキュメントをアップロードし、メタデータとして所属部署を付与する。

## 3. 非機能要件
- **性能**: 検索 1.5s以内, 生成 5.0s以内 (p95)。
- **可用性**: 99.9% 以上。
- **同時接続**: 最大 100名。

## 4. セキュリティ・制約事項
- **ネットワーク**: インターネット経由のアクセス禁止。PSC エンドポイントを唯一の経路とする。
- **暗号化**: GCS, Vertex AI Search における CMEK (Customer Managed Encryption Keys) の適用。
- **API制約（遵守規則）**:
    - `data_store_id` / `engine_id`: `^[a-z0-9]([a-z0-9-_]{0,61}[a-z0-9])?$` (Google Cloud Discovery Engine 仕様準拠)
    - メタデータキー名: `^[a-zA-Z][a-zA-Z0-9_]{0,63}$`
