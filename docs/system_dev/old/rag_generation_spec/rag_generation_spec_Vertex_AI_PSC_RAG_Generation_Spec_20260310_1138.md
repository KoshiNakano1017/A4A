# RAG生成仕様：Vertex AI PSC RAG

## 1. 生成方針 (Grounding Policy)
- **グラウンディング**: 必ず提供された検索結果（コンテキスト）のみに基づいて回答を生成する。
- **根拠不足時の対応**: 検索結果に回答が含まれない場合は、憶測で答えず「提供された資料の中には該当する情報が見つかりませんでした」と回答すること。
- **言語**: 日本語で回答する。

## 2. プロンプトテンプレ (Prompt Template)
```text
あなたは社内ドキュメントに基づく専門のアシスタントです。
以下の【コンテキスト】の内容のみを使用して、ユーザーの【質問】に正確に答えてください。

【コンテキスト】
${contexts}

【質問】
${query}

【回答ルール】
1. 回答は簡潔にまとめてください。
2. 回答の根拠となった箇所には、[1], [2] のように引用番号を付与してください。
3. コンテキストに情報がない場合は「情報が見つかりませんでした」と答えてください。
```

## 3. 出力契約 (Output Contract)
API `/api/ask` の Response JSON 内の `answer` および `citation` 配列の形式。

```json
{
  "answer": "2024年度の予算編成方針はデジタル化の推進です[1]。",
  "citations": [
    {
      "id": 1,
      "source_title": "2024_Budget_Plan.pdf",
      "uri": "gs://bucket/2024_Budget_Plan.pdf",
      "relevant_text": "デジタル化の推進を最優先課題とする。"
    }
  ]
}
```

## 4. ストリーミング仕様
- **採用有無**: **不採用** (初回リリースは一括返却のみ。UX向上のため将来的に検討 TBD)。

## 5. 安全性 (Safety & Guardrails)
- **PII (個人情報) フィルタ**: Cloud DLP API を活用し、生成 AI の回答に含まれる可能性のある個人情報をマスクする。
- **プロンプトインジェクション対策**: 入力クエリの長さを制限（最大 500文字）し、システムプロンプトの改ざんを試みるキーワードをバリデーションで検知する。
