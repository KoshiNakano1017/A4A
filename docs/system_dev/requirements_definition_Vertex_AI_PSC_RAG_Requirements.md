# 要件定義書：Vertex AI PSC RAG & API連携基盤

## 用語定義
- **PSC**: Private Service Connect。Google Cloud のサービスを VPC 内部からプライベートに利用するためのネットワーク技術。

## 1. 目的
本システムは、セキュアな閉域網（PSC）環境下で Vertex AI Search を活用した RAG（Retrieval-Augmented Generation）機能を提供するとともに、多様な外部/内部APIからのデータを標準化・整形して連携する基盤を構築することを目的とする。

## 2. スコープ
- **閉域RAG基盤**: PSC 経由での検索・生成、CMEK によるデータ保護。
- **データ連携・整形基盤**: Application Integration と Cloud Functions を活用したデータ変換・オーケストレーション。
- **認可・管理機能**: 部署（department_id）単位のアクセス制御、ドキュメント管理。

## 3. 主要ユースケース
- **一般ユーザー**: 自身の所属部署に許可されたドキュメント群に対し、自然言語で質問し回答を得る。
- **管理者**: 部署ごとのドキュメントアップロード、メタデータ（department_id等）の付与。
- **システム連携**: 外部APIからのデータを自動整形し、RAG用データストアまたは後続システムへ連携する。

## 4. 機能要件
### 4.1 RAG・検索機能
- **閉域接続**: インターネットを経由せず、Google Cloud 内部の PSC エンドポイントを通じて Vertex AI Search にアクセスする。
- **部署フィルタリング**: 検索時にユーザーの `department_id` をフィルタ条件として強制し、他部署の情報を遮断する。
- **根拠提示**: 回答の根拠となったドキュメントの引用（Citation）を表示する。

### 4.2 データ連携・整形機能
- **ローコード連携**: Application Integration をハブとし、GUI ベースでデータフローを定義する。
- **カスタムロギング/整形**: 複雑な変換ロジックは Cloud Functions (2nd gen) で実装し、Integration から呼び出す。
- **スキーマ変換**: JSON/XML/CSV 等の異なる形式を、AIエージェント等が解釈可能な標準形式に変換する。

### 4.3 認可・管理
- **部署管理**: `department_id` に基づくマルチテナント的なデータ分離。
- **ドキュメント管理**: GCS へのアップロードと、Vertex AI Search へのインデックス登録・メタデータ同期。

## 5. 非機能要件
- **性能 (SLO)**:
    - 検索・生成レスポンス（一括回答）: **5.0s 以内 (p95)**
    - データ整形（Application Integration）: 2.0s 以内 (p95, 変換処理のみ)
- **可用性**: 99.9% 以上。
- **セキュリティ**:
    - **ネットワーク**: インターネット経由のアクセスを排除。
    - **暗号化**: GCS, Firestore, Vertex AI Search において CMEK を適用。
    - **監査**: 全てのリクエストログ（誰が、いつ、どの部署のデータにアクセスしたか）を Cloud Logging に記録。

## 6. 制約事項
- **API命名規則**: Google Cloud Discovery Engine 仕様に準拠。
    - `data_store_id`: `^[a-z0-9]([a-z0-9-_]{0,61}[a-z0-9])?$`
- **メタデータ**: 1ドキュメントあたりのメタデータ数は公式制限内とする。

## 8. 受入条件
- 閉域網内から RAG 検索・回答が可能であること。
- 異なる `department_id` を持つユーザー間で、検索結果が正しく分離されていること。
- Application Integration 経由でサンプルデータのフォーマット変換が成功すること。
