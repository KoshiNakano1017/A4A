# old_versions/

v1 / v2 / 非PSC など、正本でなくなった設計ファイルの隔離先です。

- **正本**は `docs/system_dev/` 直下に役割名のみのファイル名で1種類1ファイル（例: `er_diagram_RAG_Application_Data_Model.md`, `flow_diagram_Vertex_AI_PSC_RAG.md`）。
- バージョン管理は **Git** で行い、ファイル名に v3 や Final を付けない方針です。
- 古い版はここへ移動し、正本との混在による実装ミスを防ぎます。

## 環境整理について（手動で実施）

アーキテクトエージェントに付与されている `write_design_doc` ツールは、**指定パスへの書き込み**および**上書き時の archive 退避**のみを行います。フォルダの新規作成（本フォルダ含む）や、既存ファイルの一覧・移動・削除はツールに含まれていません。

**docs/system_dev 配下の古いバージョン付きファイルを整理する場合**は、ユーザーまたは PM が手動で、またはファイル操作権限を持つスクリプト・エージェントで、以下を実施してください。

1. 必要に応じて `docs/system_dev/old_versions/` を作成する。
2. 正本でないファイル（v1, v2 等の別名や、正本と役割が重複する古い版）を `old_versions/` へ移動する、または削除する。
3. 正本（役割名のみのファイル名）は `docs/system_dev/` 直下に残す。

アーキテクトは正本の維持・更新に専念し、一括整理は行いません。
