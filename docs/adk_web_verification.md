# ADK Web から動作確認する方法

ADK の Web UI および A4A の Web UI を使って、エージェントの動作をブラウザから確認する手順です。

---

## 前提

- プロジェクトルートで `uv sync` 済み
- `.env` に `GEMINI_API_KEY`（または `GOOGLE_API_KEY`）を設定済み

---

## 方法1: ADK Web（単一 or 複数エージェントを切り替えて対話）

Google ADK 標準の Web インターフェースです。エージェントを選んでチャットで対話できます。

### 起動

**プロジェクトルート**で実行します（エージェントを含むディレクトリの親で実行する必要があります）。

```bash
uv run adk web
```

`adk` が PATH にない場合も、`uv run` でプロジェクトの仮想環境内の `adk` が使われます。

- デフォルトで **http://localhost:8000** で起動します。
- ポートを変える場合:
  ```bash
  uv run adk web --port 8080
  ```
  → **http://localhost:8080** でアクセスします。

### 注意: ポートの競合

- **`make run` / `uv run python -m a4a.run_all`** を使うと、**Coordinator が 8000** で起動します。
- **ADK Web もデフォルトで 8000** を使うため、**両方を同時に使う場合は ADK Web のポートを変更**してください。
  ```bash
  uv run adk web --port 8080
  ```
- **ADK Web だけ**で動作確認する場合は、そのまま `uv run adk web` で問題ありません。

### 動作確認の流れ

1. ブラウザで **http://127.0.0.1:8000**（または `--port` で指定したポート）を開く。
2. 画面上で **エージェントを選択**する（左上などのドロップダウンで、プロジェクト内のエージェントが一覧になる想定）。
3. チャット欄にメッセージを入力して送信する。
4. 応答がストリーミング表示され、**Event history / State** などで実行の流れを確認できる。

### ADK Web でできること

- **チャット**: 選択したエージェントと対話
- **Event history**: エージェント実行時のイベント確認
- **State**: セッション状態の確認・編集（開発・デバッグ用）
- **Session 管理**: セッションの切り替え

> **注意**: ADK Web は開発・デバッグ用です。本番運用には使いません。

## 方法2: A4A Web UI（Coordinator 経由でルーティング）

A4A 専用の Web UI（`a4a.web`）です。**Coordinator（PM）を経由**して、クエリに応じたエージェントにルーティングし、**どのエージェントが応答したか**を表示します。

### 起動

**まず全エージェント＋Coordinator を起動**してから、別ターミナルで Web UI を起動します。

**ターミナル1**（全エージェント + Coordinator）:

```bash
uv run python -m a4a.run_all --verbose
```

**ターミナル2**（Web UI）:

```bash
uv run python -m a4a.web
```

または:

```bash
make ui
```

- **http://localhost:8888** で起動します（`WEB_PORT` で変更可能）。

### 動作確認の流れ

1. ブラウザで **http://localhost:8888** を開く。
2. チャット欄にクエリを入力して送信する。
3. 応答の**先頭に `[AGENT:エージェント名]`** が表示され、**どのエージェントが処理したか**が分かります。
4. PM が複数エージェントに依頼した場合は、表示されるのは主にルーティング先の 1 つです。

### A4A Web UI の特徴

- **Coordinator 経由**: ユーザーは 8888 に送るだけで、PM が適切なサブエージェントに振り分ける。
- **誰が応答したか**: ストリーミング応答に `[AGENT:○○]` が付く。
- **run_all 必須**: 8000 で Coordinator と各サブエージェントが動いている必要がある。

---

## 比較

| 項目 | ADK Web (`adk web`) | A4A Web UI (`make ui` / `a4a.web`) |
|------|---------------------|-------------------------------------|
| 起動コマンド | `uv run adk web` | 先に `run_all`、別ターミナルで `uv run python -m a4a.web` または `make ui` |
| ポート | 8000（`--port` で変更可） | 8888 |
| 対話の経路 | 選択した 1 エージェントと直接対話 | Coordinator 経由で 1 エージェントにルーティング |
| エージェント選択 | 画面上でドロップダウン等で選択 | クエリ内容で PM が自動選択 |
| 用途 | 単体エージェントの開発・デバッグ | 全体フロー（PM ルーティング）の確認 |

## トラブルシューティング

### 「Python was not found」/「adk が認識されない」

- コマンドは **`uv run adk web`** または **`uv run python -m a4a.web`** のように **`uv run` を先頭に付けて**実行してください。
- 仮想環境を有効にしている場合は `adk web` だけでも動くことがあります。

### ポート 8000 が使えない

- Coordinator と同時に ADK Web を使う場合: `uv run adk web --port 8080` で別ポートにしてください。
- A4A Web UI は 8888 のため、run_all（8000）と競合しません。

### Web UI で応答が返らない

- A4A Web UI を使う場合は、**ターミナル1で `run_all` が起動したまま**か確認してください。
- `.env` の API キーが有効か、`docs/troubleshooting.md` の「GEMINI_API_KEY」の項を確認してください。
