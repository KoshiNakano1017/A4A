# テスト仕様書の格納先

このディレクトリには **テスト仕様書** および関連するテスト結果メモを格納します。

---

## 格納するもの

- **テスト仕様書**: 項目・手順・期待結果・前提条件をまとめたドキュメント
- **テスト結果メモ**: 実施日・実施者・OK/NG・備考（任意）
- **動作確認チェックリスト**: リリース前や変更後の確認項目一覧（任意）

---

## ファイル名の例

- `system_dev_tool_verification.md` … システム開発ツールの動作確認仕様
- `a4a_basic_verification.md` … A4A 基本動作（起動・クエリ・Web UI）の確認仕様
- `YYYY-MM-DD_verification_result.md` … 実施日付の結果メモ

---

## 動作確認手順との関係

- 手順の詳細は **`../verification_procedure.md`** を参照する
- ここに格納した仕様書に沿って確認を実施し、結果を同じディレクトリにメモしてもよい
