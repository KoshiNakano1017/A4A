## 実装・セキュリティ制約（重要）
- **依存ライブラリの管理**: `requirements.txt` を作成する際は、必ずバージョンを固定すること（例: `pandas==2.2.2`）。最新版をそのまま記述せず、既知の安定版を指定する。
- **パス操作の安全性**: ユーザーが指定したパスを扱う際は、必ず `os.path.abspath` や `os.path.normpath` を使用し、意図しないディレクトリ（親ディレクトリ等）へのアクセス（パス・トラバーサル）を防ぐバリデーションを実装すること。
- **正規表現の処理**: 外部（設定ファイル等）から読み込んだ正規表現を `re.compile` する際は、エラーハンドリングを徹底し、可能な限り複雑な量指定子（ReDoSのリスク）に対する警告や制限を考慮すること。
- **システム操作**: `os.startfile` 等でファイルを開く際は、対象がプログラム内で生成した特定の拡張子（.txt, .csv 等）であることを確認してから実行するガード節を入れること。

3. コード生成前に、読み取った要件と上記「実装・セキュリティ制約」を照らし合わせたチェックリストを内部で作成すること。

ユニットテストの導入: src/logic/ 配下の各ロジック（加工ルール、バリデーション）に対する自動テストを追加することで、今後の機能拡張時のデグレを防げます。
配布用ビルドスクリプト: PyInstaller を使用した .exe 化の工程を自動化する build.py の作成。
多言語対応: 現在 GUI のラベルが一部英語と日本語が混在しているため、リソースファイルによる文言管理への移行。

概要: ExcelやYAML形式の設定ファイルに基づき、ローカルのCSV/TSVファイルを読み込み、加工・変換・フィルタリングを行うPythonアプリケーション（GUI版）です。セキュリティと保守性に配慮し、外部通信を行わない設計となっています。
生成ファイル（OUTPUT_PROJECT_ROOT 配下）
requirements.txt: 依存ライブラリの定義（バージョン固定済）
src/main.py: アプリケーションの起動用エントリーポイント
src/gui/app.py: TkinterによるGUI実装（進捗表示・キャンセル機能・安全なログ表示）
src/logic/config_loader.py: 設定ファイルの読み込みとReDoS対策を含むバリデーション
src/logic/validator.py: 入出力パスの妥当性チェックとパス・トラバーサル防止機能
src/logic/processor.py: Pandasを用いたCSV加工ロジック（型変換・フィルタリング・列マッピング）
src/utils/logger.py: 処理詳細を記録するログ管理クラス
使い方・次に調整できる点
使い方:
pip install -r requirements.txt を実行して、必要なライブラリをインストールしてください。
python src/main.py を実行してアプリケーションを起動します。
GUI上で「単一ファイル」または「フォルダ一括」を選択し、加工対象と設定ファイル、出力先を指定して「実行」ボタンを押してください。
調整できる点:
変換ルールの追加: src/logic/processor.py の _apply_conversions メソッドに新しい処理を追加することで、独自の変換（特定の計算式など）を容易に拡張できます。
配布用パッケージ化: PyInstaller を使用することで、PythonがインストールされていないWindows環境向けの .exe ファイルを作成可能です。
バリデーションの強化: 設定ファイル内の特定の値の範囲チェックなど、業務に特化した事前チェック機能を追加できます。
